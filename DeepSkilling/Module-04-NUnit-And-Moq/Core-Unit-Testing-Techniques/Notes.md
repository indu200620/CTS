# Core Unit Testing Techniques

## Testing Strings

Verify string operations such as:

* Concatenation
* Length
* Formatting

## Testing Arrays and Collections

Verify:

* Element count
* Search operations
* Sorting results

## Testing Return Types

Confirm methods return expected values and data types.

## Testing Void Methods

Validate state changes after execution.

## Testing Exception Handling

Verify exceptions using Assert.Throws.

Example scenarios:

* Invalid input
* Null values
* Divide by zero

## Testing Private Methods

Generally test through public methods rather than directly accessing private methods.

## Code Coverage

Code coverage measures how much application code is executed during testing.

Benefits:

* Identifies untested code
* Improves software reliability
* Increases confidence during deployment
